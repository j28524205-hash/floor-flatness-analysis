# PRD: 바닥면 평탄성 분석 웹 앱 (floor_flatness_app.py)

> 본 문서는 `평활도측정.md`(개발/수정 이력 문서)를 기준으로 작성된 제품 요구사항 정의서(PRD)입니다.

---

## 1. 배경 및 문제 정의

- 건축 마감면(바닥) 품질 검사는 전통적으로 수작업(자·수평계)에 의존해 노동집약적이고 검사자 간 편차가 크다.
- 점군 데이터를 활용한 자동화 연구는 많지만, 대부분 고가의 전문 장비·소프트웨어에 의존하며 현장에서 결과를 즉시 확인하거나 보고서로 출력하는 기능이 부족하다(`한일심포지엄` 논문의 문제의식과 동일).
- 이를 해결하기 위해 오픈소스 Python 스택만으로 동작하는 로컬 웹 앱을 개발하여, 점군 데이터 업로드 → 자동 분석 → 시각화/등급 판정 → 보고서 출력까지 한 번에 처리한다.

## 2. 목표 (Goals)

1. 점군 데이터로부터 바닥면의 평탄도(정상/돌출/함몰/균열)를 **자동·정량적으로 측정**한다.
2. 분석 결과를 **웹 UI로 즉시 시각화**하고, **TXT/PDF 보고서**로 자동 출력한다.
3. 전문 장비·소프트웨어 없이 **오픈소스 스택만으로** 동작한다.
4. 다양한 스캔 조건(노이즈 수준, 바닥 외 수평 구조물 포함 여부, 좌표 단위 등)에서도 **크래시 없이 견고하게** 동작한다.
5. 논문(`한일심포지엄_영문초안.md`)에 서술된 알고리즘과 **코드가 항상 일치**하도록 유지한다.

## 3. 범위 (Scope)

### In Scope
- 점군 전처리(SOR 이상치 제거) 및 바닥/벽체 분리(OR/AND 두 방식 선택 가능)
- RANSAC 기반 기준 평면 추정(인라이어 전체 최소제곱 재적합 포함) 및 최소 인라이어 비율 검증
- 법선 벡터 기반 균열 탐지(곡률/불연속성/비등방성 가중 결합)
- KCS 41 46 01(±3mm) 기준 4등급(A/B1/B2/C) 자동 분류
- 2D 히트맵, 3D 표면/산점도, 히스토그램 시각화 (보간 실패 시 자동 대체)
- 좌표 단위(m/mm) 명시적 선택 및 자동 환산
- 결과 CSV/TXT/PDF 자동 출력, 사용된 파라미터의 보고서 내 투명한 기록
- 로컬 Streamlit 실행(.bat 더블클릭)

### Out of Scope (현 단계 기준)
- 실제 현장 점군 데이터를 통한 검증 (현재는 함수 단위 테스트로만 검증됨, 10절 "알려진 제한사항" 참고)
- 웹 배포(Cloud Run, Streamlit Community Cloud 등) — 별도 트랙
- 다양한 마감재 유형(벽체, 타일, 도장면 등)에 대한 일반화
- 실시간 연속 모니터링
- 크랙 점수의 파일 간 절대 비교(현재는 스캔 내부 상대 지표로만 제공)

### 관련 레거시 파일
- `floor_flatness_analysis.ipynb` — Google Colab용 초기 프로토타입 노트북(Open3D 기반, DBSCAN 결함 클러스터링 포함). 현재 `floor_flatness_app.py`로 대체되어 **미사용** 상태이며, `바닥면_평탄성_분석_실행.bat`도 이 노트북을 참조하지 않음.

## 5. 기능 요구사항 (Functional Requirements)

### 5.1 알고리즘 파이프라인 (`run_analysis()`)
1. **이상치 제거(SOR)** — k=20 이웃, 표준편차 배수(기본 2.0, 조정 가능). 제거 예상 비율이 30% 초과 시 안전장치 발동, 원본 유지 + 상태값(`ok`/`no_outliers`/`skipped_too_many`) 반환.
2. **바닥/벽체 분리** — Z 여유값 + 법선 각도 필터. **OR(기본, 논문 기준)** 또는 **AND(엄격)** 모드 선택 가능.
3. **기준 평면 추정(RANSAC)** — 반복 횟수(기본 1,000회, 조정 가능)로 최적 3점 평면을 찾은 뒤, **인라이어 전체를 최소제곱 재적합**해 정확도 향상. 최소 인라이어 비율(기본 30%, 조정 가능) 미달 시 명확한 에러로 중단.
4. **균열 탐지** — k=15 이웃 공분산 고유값 분해, 곡률 40%+법선 불연속성 40%+비등방성 20% 가중 결합. *(주의: 정규화가 스캔 내부 상대 기준이므로 파일 간 절대 비교 불가 — UI/보고서/CSV 컬럼명에 명시)*
5. **포인트 분류** — KCS 41 46 01(±3mm) + 균열 점수로 A/B1/B2/C 4등급 자동 분류.

### 5.2 데이터 입력
- CSV/TXT/XYZ 업로드, 헤더 유무·구분자 자동 감지
- 컬럼 수(3개 미만), NaN/Inf 검증 및 자동 제외
- **좌표 단위(m/mm) 명시적 선택** — mm 선택 시 자동 1000분의 1 환산, m 선택 시에도 좌표 범위 이상 시 재확인 경고
- 샘플 데이터(바닥+벽체 합성) 제공
- 최소 포인트 수(30개, `MIN_POINTS`) 검증은 **2단계**로 이뤄짐: ① 업로드 직후 원본 포인트 수 기준, ② `run_analysis()` 내부에서 이상치 제거 후 남은 포인트 수 기준(4차 수정에서 추가)

### 5.3 고급 설정 (UI 노출 파라미터)
- RANSAC 인라이어 허용오차(mm), 최소 인라이어 비율(%), 반복 횟수
- 이상치 제거 민감도(표준편차 배수)
- 바닥 분리 방식(OR/AND) — 항상 보이는 위치에 노출(고급 설정 아님)

### 5.4 시각화
- 2D 편차 히트맵, 3D 표면(Z축 과장 ×1~×500), 등급별 3D 산점도, 편차 히스토그램
- `griddata` 선형 보간 실패 시 **nearest로 자동 대체** + 사용자에게 캡션으로 고지. nearest마저 실패하면(극히 드문 경우) 기존처럼 경고 표시 후 해당 시각화만 건너뜀

### 5.5 결과 출력
- 종합 등급(우수/양호/보통/불량) 판정
- 결과 CSV(컬럼명에 상대 지표 여부 명시), TXT 요약, PDF 보고서(그래프 포함)
- 보고서에 사용된 모든 파라미터(임계값, 좌표 단위, 이상치 제거 비율, 바닥 분리 방식 등) 투명하게 기록

## 6. 비기능 요구사항 (Non-Functional Requirements)

- **견고성**: 퇴화 데이터(일직선 배치, 이상치 과다, 소규모 점군 등)에서 크래시 대신 명확한 에러 메시지로 안내
- **재현성**: RANSAC은 시드 고정(기본 42)으로 동일 입력 시 동일 결과 보장. *(단, 3D 시각화의 5만 점 초과 서브샘플링(`vis_idx`)은 아직 시드 미고정 상태로 남아있어 분석 결과 자체가 아닌 화면 표시 포인트만 매번 달라질 수 있음 — 낮은 우선순위로 보류된 항목)*
- **이식성**: PDF 생성 시 OS별 한글 폰트 자동 탐색(Windows/Linux/macOS) — 향후 클라우드 배포 대비
- **일치성**: 알고리즘 파라미터·부호 규약이 논문(`한일심포지엄_영문초안.md`)과 항상 일치
- **비용**: 오픈소스 Python 스택(numpy, scipy, scikit-learn, Streamlit, Plotly, fpdf2)만으로 구현

## 10. 알려진 제한사항

- `separate_floor`의 OR 결합 방식은 바닥 외 다른 수평면(테이블 상판, 계단 디딤판 등)을 바닥으로 오분류할 수 있음. AND 모드로 완화 가능하나 **기본값은 논문 일치성을 위해 OR 유지** (반복 지적되었으나 유지하기로 확정).
- 크랙 점수는 **스캔 내부 상대 정규화**이므로 서로 다른 파일 간 절대 비교 불가 (UI/보고서/CSV에 명시, 알고리즘 변경 없음).
- `separate_floor`가 포인트별 Python 반복문으로 SVD를 수행해 대규모 점군(수십만 점 이상)에서 느릴 수 있음(성능 최적화 미착수).
- **실측 검증 데이터 없음, 그리고 앱 자체의 검증과 논문 수치 생성은 서로 다른 경로임**:
  - 논문(`한일심포지엄_영문초안.md`) Figure 2·3 및 통계치(불량률 12.9% 등)는 `floor_flatness_app.py`가 아니라 **별도의 독립 스크립트**(`한일 심포지엄/figures/` 생성용, 앱과 코드가 분리됨)로 만든 가상 데이터 기준임.
  - `floor_flatness_app.py` 자체는 대칭 링, 퇴화(일직선) 데이터, 이중 밀도 데이터 등을 이용한 **함수 단위 테스트**로만 검증됨(1~8차 수정 이력의 "검증" 항목 참고). 논문과 동일한 가상 바닥 데이터셋으로 앱을 직접 검증한 적은 없음.

- **목적**: 본 PRD(특히 5절 기능 요구사항)를 바탕으로 만든 **정적 UI 스크린 목업**. `floor_flatness_app.py`(Streamlit 실행 앱)와 **연동되지 않음** — 버튼 클릭, 파일 업로드 등 실제 분석 기능은 전혀 동작하지 않는다.
- **구현 방식**: 순수 HTML/CSS/JS 단일 파일(외부 프레임워크 없음). Google Fonts CDN(`Big Shoulders`, `IBM Plex Mono`, `IBM Plex Sans KR`)만 외부 의존성으로 사용하며, 그 외 전부 인라인 `<style>`/`<script>`로 자체완결(self-contained).
- **디자인 컨셉**: 건설 도면/블루프린트(blueprint) 스타일의 테크니컬 엔지니어링 톤. 어두운 네이비 배경 + 청록(cyan) 격자선, 패널 모서리에 도면 주석풍 브래킷(corner bracket), 상단에 스캔라인 애니메이션(점군 스캐닝 은유). 4등급(A/B1/B2/C) 색상 코드: 정상=초록, 돌출=주황, 함몰=파랑, 크랙=핑크/레드.
- **재현된 화면 구성** (PRD 5절 순서와 1:1 대응):
  1. 데이터 입력(드롭존, m/mm 단위 세그먼트, 샘플 데이터 버튼)
  2. 판정 기준 & 바닥 분리 설정(OR/AND 토글, 경고 캡션, 고급 설정 `<details>` 접이식 패널 — RANSAC 인라이어 허용오차/최소 인라이어 비율/반복횟수/SOR 민감도)
  3. 분석 결과 요약(총 입력점/벽체제거/바닥분석점 메트릭 + A/B1/B2/C 등급 타일 + 종합등급 배너)
  4. 2D 히트맵 — JS로 24×18 그리드를 생성하며 대각선 방향에 고편차(크랙) 스트라이프를 의도적으로 배치해 논문 Figure 2의 시각적 톤(대각선 균열 패턴)과 맞춤
  5. 편차 히스토그램 — 정규분포형 바 + ±3mm 기준선
  6. 3D 시각화 — CSS `transform: rotateX/rotateZ`로 만든 의사(疑似) 3D 격자 위에 등급별 색상 점 무작위 배치, 탭 3개(3D 표면/분류별 3D/전체 포인트클라우드, 탭 전환은 활성 스타일만 바뀌고 내용은 동일 — 실제 뷰 전환 기능 없음)
  7. 데이터 테이블(예시 5행)
  8. 종합 결과 보고서 미리보기(실제 `report_text` 포맷 재현: 입력/처리 좌표 단위, RANSAC 설정, 바닥 분리 방식, 이상치 제거 비율 등 8차까지의 투명성 필드 반영) + CSV/TXT/PDF 저장 버튼(비활성, 시각적 목업만)
- **표시된 수치는 전부 가상/예시 값**(총 입력점 45,231개, A 87.1%/B1 4.2%/B2 6.9%/C 1.8%, 불량률 12.9% 등)이며, 논문 3.2절에 보고된 예시 통계치와 의도적으로 맞춤(일관된 스토리텔링을 위함). 실제 분석 결과가 아님.
